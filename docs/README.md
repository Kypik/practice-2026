# Процесс выполнения практики



В рамках проектной практики была поставлена задача разработки статического веб-сайта для представления проекта «Система обеспечения автономного управления транспортными платформами».



### Основные этапы выполнения:



1. **Подготовительный этап (февраль - март)**

   - Изучение требований к практике и материала по Git.

   - Постановка целей и задач.



2. **Разработка структуры репозитория (начало марта)**

   - Создание корректной структуры Git-репозитория согласно требованиям (`site/`, `docs/`, `reports/`).



3. **Создание статического сайта (март - апрель)**

   - Изучение HTML и CSS.

   - Разработка всех обязательных страниц: Главная, О проекте, Участники, Журнал, Ресурсы.

   - Реализация дизайна и адаптивной вёрстки.

   - Наполнение сайта текстовой информацией, изображениями и скриншотами.



4. **Ведение документации (апрель)**

   - Ведение журнала выполнения работ.

   - Описание ИТ-мероприятия.

   - Подготовка материалов для итогового отчёта.



5. **Выполнение вариативной части**
   - Выбор технологии для практической реализации.

   - Изучения материала по выбранной технологии.
   - Реализация технологии.



6. **Подготовка итогового отчёта**

   - Заполнение шаблона отчёта по практике.

   - Оформление отчёта в форматах .docx и .pdf.



## Результаты практики



   - Создан полноценный статический сайт проекта с современным дизайном.

   - Ведётся вся необходимая документация в формате Markdown.

   - Реализована выбранная технология в рамках вариативной части.

   - Подготовлен итоговый отчёт по практике.
